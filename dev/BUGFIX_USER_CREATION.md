# Bug Fix: Ubuntu User Creation Failure

## Problem Reported

After restarting, the installer:
1. Tries to install Ubuntu again and again
2. Times out waiting for Ubuntu installation
3. Fails to set Ubuntu as default
4. ERROR: Failed to create Ubuntu user
5. Installation aborts

## Root Cause

When Ubuntu is freshly installed via `wsl --install -d Ubuntu`, there is **no default user configured**.

The installer was trying to run:
```nsis
wsl -d Ubuntu -u root -- bash -c "bash /path/to/script.sh"
```

But without a default user, WSL commands would:
- Prompt for interactive user setup (which hangs in automated installer)
- Or fail completely because no user context exists

## Solution Applied

### Key Changes:

1. **Set root as default user FIRST** using `ubuntu.exe config`:
   ```nsis
   ubuntu.exe config --default-user root
   ```
   This works even when no user exists yet. Includes fallback to full WindowsApps path if needed.

2. **Check if user already exists** before attempting creation:
   ```nsis
   wsl -d Ubuntu -u root -- id $ConfigUsername
   ```
   Prevents errors from recreating existing users.

3. **Run user creation commands** directly via wsl.exe as root:
   ```nsis
   wsl -d Ubuntu -u root -- useradd -m -s /bin/bash $ConfigUsername
   wsl -d Ubuntu -u root -- bash -c "echo '$ConfigUsername:$ConfigPassword' | chpasswd"
   wsl -d Ubuntu -u root -- usermod -aG sudo $ConfigUsername
   ```
   Now these work because root is the default user.

4. **Configure wsl.conf and ubuntu.exe** with the new user:
   ```nsis
   wsl -d Ubuntu -u root -- bash -c "echo -e '[user]\ndefault=$ConfigUsername' > /etc/wsl.conf"
   ubuntu.exe config --default-user $ConfigUsername
   ```

5. **Added comprehensive error handling** with informative messages and user prompts.

### Updated Flow:

```
1. Shutdown WSL
2. Set root as default: ubuntu.exe config --default-user root (with fallback)
3. Check if user exists (skip creation if yes)
4. Create user account: useradd
5. Set password: chpasswd
6. Add to sudo group: usermod
7. Configure sudoers
8. Set wsl.conf default user
9. Set ubuntu.exe default user: ubuntu.exe config --default-user username
10. Verify: wsl -d Ubuntu -- whoami
```
